# PatternFlow ESP32 — 하드웨어 종합 참고자료

## 보드 스펙

| 항목 | 스펙 |
|------|------|
| 모듈 | ESP32-S3-WROOM-1 |
| 변형 | **N16R8** (Flash 16MB + PSRAM 8MB) |
| BLE | 내장 (BLE 5.0) |
| WiFi | 내장 (2.4GHz) |
| 코어 | 듀얼 코어 Xtensa LX7 |

> N16R8은 최상위 모델. HUB75E DMA (~150KB) + NimBLE (~70KB) 동시 구동해도 PSRAM 여유 7MB 이상.

---

## BOM (부품 목록)

| 부품 | 수량 | 비고 |
|------|------|------|
| ESP32-S3-DevKitC-1 (N16R8) | 1 | 듀얼코어, BLE 5.0 내장 |
| 128×64 P2.5 HUB75E LED 매트릭스 | 1 | 알리 ~27,000원 |
| 10KΩ 포텐셔미터 (노브) | 4 | 기능은 소프트웨어로 정의 |
| 5V 전원 어댑터 (4A 이상) | 1 | LED + ESP32 공용 |
| 보조배터리 (5V 출력) | 1 | 휴대용 구동 시 |
| 케이스 | 1 | 3D 프린팅 (PLA), 보조배터리 거치대 내장 |

---

## 전원

```
[5V 외부 전원] ──┬──▶ LED 매트릭스 (5V, 최대 ~4A)
                 └──▶ ESP32-S3 (5V, Vin)
```

ESP32 + 매트릭스가 같은 5V 라인에 묶여 있음. BLE 운영 시 USB 불필요 → 이중 공급 문제 없음.

# V1 하드웨어 (현재 — 참고용 아카이브)

> ⚠️ 아래는 V1 브레드보드/수작업 배선 기준. V2에서 PCB로 대체 예정.

## V1 노브 배치 (정면 기준)

```
┌───────────────────┐
│  Knob4(⚡)  Knob3(🎲) │
│  Q2/Speed   Q1/Pattern │
│                       │
│  Knob1(🎨)  Knob2(🔀) │
│  Q3/Hue     Q4/Mode   │
└───────────────────┘
```

| 4  | GND | GND |

| 7  | B2  | 13  |

| 14 | LAT | 47  |

### 노브 (포텐셔미터 4개)

| 노브 | 기능 | 가운데 → GPIO | 양쪽 다리 |
|------|------|-------------|----------|
| Knob1 (Q3-좌하) | Hue 🎨 | GPIO 4 | 3V3 + GND |
| Knob2 (Q4-우하) | Mode 🔀 | GPIO 5 | 3V3 + GND |
| Knob3 (Q1-우상) | Pattern 🎲 | GPIO 6 | 3V3 + GND |
| Knob4 (Q2-좌상) | Speed ⚡ | GPIO 7 | 3V3 + GND |

### V1 GPIO 총 사용량

| 용도 | GPIO 핀 | 개수 |

| HUB75E 행선택 (A,B,C,D,E) | 45, 11, 48, 12, 21 | 5 |

## V1 알려진 이슈

- **노브 불안정**: 납땜 접촉불량으로 인해 노브 값이 튀는 현상 발생. V2에서 PCB로 해결 예정.

# V2 하드웨어 (확정)

## 목표

V1의 수작업 배선을 **PCB 기반**으로 전환. 납땜 접촉불량 제거 + 조립 편의성 확보.

## V2 GPIO 매핑

### HUB75E → ESP32-S3

| HUB75 핀 | 신호 | ESP32 GPIO | 비고 |
|----------|------|-----------|------|
| 1  | R1  | 42  | |
| 2  | G1  | 41  | |
| 3  | B1  | 40  | |
| 4  | GND | GND | |
| 5  | R2  | 38  | |
| 6  | G2  | 39  | |

| 11 | C   | 16  | V1: 48 → 16 |
